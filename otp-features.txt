Purpose
- This file separates OTP, TOTP, backup-code, and authentication-code security features from the broader FlowChat CRM and chat feature set.
- FlowChat currently implements app-based TOTP two-factor authentication and backup-code recovery. SMS/email OTP is not implemented in the current codebase, but can be treated as a planned authentication extension if required by the product roadmap.

Signin Flow With 2FA
- User submits email and password.
- Password is verified with Argon2.
- If TOTP is not enabled, a session token is created immediately.
- If TOTP is enabled, the API returns a requiresTwoFactor response.
- Client must collect an authenticator code or backup code.
- Client submits the second factor with the user ID.
- API verifies the authenticator code or backup code.
- If verification succeeds, the API creates a session token.
- If a backup code is used, the used backup code is removed or invalidated.
- If verification fails, signin does not create a session.

Stored OTP/2FA Data
- User TOTP secret.
- TOTP enabled timestamp.
- Hashed backup codes.
- User ID used during intermediate 2FA signin.
- Session token created only after successful second-factor verification.

Security Properties
- TOTP uses authenticator-app codes rather than reusable static codes.
- Backup codes are one-time recovery codes.
- Backup codes are hashed before storage.
- Signin does not issue a session before second-factor verification when 2FA is enabled.
- 2FA disablement requires proof through TOTP or backup code.
- Backup codes are not stored as plaintext.
- Passwords are hashed with Argon2.
- Sessions are stored separately from user credentials.
